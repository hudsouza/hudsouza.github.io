Parte 4

A cascata

↑
↑

Saber como o CSS é renderizado na página é muito importante pra saber como escrever seu código. A folha de estilo funciona como uma sequência de instruções e por isso a ordem de como os seletores são declarados faz toda a diferença.

h1 {
font-size: 20px;
}

h1 {
font-size: 24px;
}

O tamanho da fonte desse h1 vai ser de 24px porque o último valor declarado sobrepõe todos os anteriores.

↑

Especificidade

↑
↑

A primeira regra para atribuir valores a propriedades é a sequência do código, mas ela pode ser sobreposta se o primeiro seletor for mais específico que o próximo. Podemos calcular a especificade com os seguintes valores:


Elemento: 0-0-1

Classe: 0-1-0

ID: 1-0-0

↑
↑

Na prática

↑
↑
.titulo {
font-size: 20px;
}
/* Especificidade: 0-1-0 */

h1 {
font-size: 24px;
}
/* Especificidade: 0-0-1 */

Apesar do segundo valor declarado ser de 24px, o valor que será atribuido será de 20px porque o seletor classe é mais específico que o seletor de elemento.

parent & child

↑
↑

Voltando um pouco para o HTML, quando estamos inserindo os elementos, muitos deles ficam dentro de outros elementos. Essa relação é chamada de parent e child

<div>
<p> ... </p>
</div>

O elemento <p> é filho (child) da div, que é elemento pai (parent)

↑
↑

pra que serve?

↑
↑

Podemos usar a relação de child e parent pra especificarmos o que vai ser estilizado.

p {
line-height: 1.5;
}

article p {
line-height: 1.8;
}

Nesse código, todos os parágrafos vão ter a entrelinha com altura 1.5, exceto as que estiverem dentro de um <article>, que terão entre linha 1.8.
